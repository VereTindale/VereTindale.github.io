blog 1 - Class and ID: A best practice guide

15/9/19

When writing the markup code for websites, classes and id's are extremely useful to identify things thta you want to syle using CSS. The problem is that, it kinda feels like they do the same thing... And they do. But, although theis doesnt seem to matter now, later versions of browsers may not recognize classes and id's as they do now, if we dont adhere to the best practice rules for them. For example: Javascript has a function called getElementById that allows you to madofy elements based on their id. It does now, however have the capaility for classes. This means that by not adhering to the best preactice, you might end up having to rebuild pages down the line.

What is best practice then?

Basically, think of the id tag as a way to identify single elements by themselves. The class tag on the other hand, identifies groups of things that you would want to change together. A good example of this is a news site; The layout tends to stay the same, but the content inside it changes almost constantly. By nesting classes within id's, we can apply things to them without effecting the boxes they are conatined in.
